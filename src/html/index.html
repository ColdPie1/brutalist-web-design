Guidelines for Brutalist Web Design

  • The content is readable on all reasonable screens and devices.
  • Only Hyperlinks and Buttons respond to clicks.
  • Hyperlinks are underlined. Buttons look like buttons.
  • The Back Button should work as expected.
  • The page scrolls as needed to allow viewing the content.
  • Basic functions of the browser and operating system work as expected.
  • Decoration only when needed.

Brutalist Web Design

The term brutalism is often associated with Brutalist Architecture, but the tenets of it as applied to architecture can be applied to other constructions. This website documents its application to website design.

An example of brutalist architecture, DC Engine Company 02's Fire Station (View full-size).

The term brutalism is derived from the French betón brut, meaning “raw concrete”, which is what most brutalist buildings are made from. It's the term raw that is most applicable to Brutalist Web Design.

In a sense, brutalism is a form of truth to materials, which means that the nature of the construction of something should not be hidden. While one apply this to web design by showing the HTML source used to create a webpage, it's more appropriate to apply the concept to the behavior and look of the web page as viewed in a web browser. This is likely how you are reading this paragraph now.

Exploring the Guidliness

The content is readable on all reasonable screens and devices.

Web pages are designed to be read. As part of being true to its nature, a web page should be readable in the ways in which it's possible to be viewed. This means a wide variety of screen sizes and even assistive technologies like screen readers, which don't have a screen at all.

Standard web technologies and techniques provide the means to do this. By using semantic markup and no styling a web site would be readonable on all screens and devices. If styling is needed, CSS provides a way to ensure that styling is applied appropriately. For example, this website ensures that line lengths are around 80 characters or less—a comfortable width for reading—on large monitors, laptops, and mobile phones.

Only Hyperlinks and Buttons respond to clicks.

Although JavaScript allows any element to respond to click events, a web page is not an application. A web page is a hypertext document that allows data to be submitted to a server via a form. To be true to its nature, a web page should only allow hyperlinks and buttons to be clicked. To allow other elements to respond to clicks is unnecessary functionality for a web page.

In particular, clicking elements such as images or undecorated text will take the user away from the content unexpectedly. No user wants this, and to do so betrays the very nature of a web page.

Hyperlinks are underlined. Buttons look like buttons.

The creation of the world-wide web popularized the hyperlink, and the convention since the dawn of the web is that links are underlined (and often set in blue text). This is a powerful convention that immediatley indicates to a user that there is more information to be found by clicking the link. Underlining links is the simplest way to stay true to the nature of a web page.

The Back Button should work as expected.

Because of the webbed nature of hypertext, web browser have included a button to go back. This button allows a user to effectively undo their last click. When the back button does not take the user back to where they were it creates confusiona nd frustration, and is fundamentally betraying the nature of a webpage and hypertext.

Fortunately, by allowing only hyperlinks and buttons to be clickable, the back button will always work. Going backafter a form submission is semantically unclear, but browser handle this. This means that it takes extra effort to break the back button.

The page scrolls as needed to allow viewing the content.

The nature of a web page is that it has content you wish to read, but that content might not fit on the screen all at once (in fact, it rarely would). With the printed page, readers flip through pages to read the content beyond what they are looking at. Because a computer screen allows scrolling content, this is the preferred method adopted by web browser.

Nevertheless, many web pages artifically paginate their content, requiring the user to both scroll down to the end of the page, and then click a hyperlink to continue reading. Some web pages even hide content that would otherwise be viewable, requiring users to click a button to reveal the content on the page they are alread in. This is a fundamental betrayal of what a web page is.

Basic functions of the browser and operating system work as expected.

The web browser is how web pages are consumed and displayed to the user. Web browser run on a variety of platforms and operating systems. These systems include features and idioms common to the operating system that users rely on to work. For example, copying text in a word processor copies the selected text to the pastboard. This should be the behavior of a web page as well.

Scrolling is another example. Scrolling content should simply move the content the same as in other applications. It should not change, reveal, or hide content. Pasting into a form should be possible and paste the contents of the pasteboard. Browser support this by default, so it takes special effort to break these conventions and idioms.

Decoration only when needed.

🧐

Web pages don't all need to look the same—they just need to stay true to what they are.

A web page is not an application. The design should serve the purpose, which is to deliver content to the user. Decoration is not strictly required to do this, but boring visual design isn't required, either. For example, this web page contains fonts of different sizes and faces. It doesn't use pure black and white as its colors. This section has a pull-quote to emphasize a point made below. It uses a blue background and a freakin' emoji.

Excessive decoration can take the user away from the web page's primary purpose or creating unintended complexities when using the web page. Web pages don't all need to look the same—they just need to stay true to what they are.

Very Few Web Sites are Applications Link Here

While a web browser and HTTP is a great platform for application delivery, most such applications are internal software or serve small audiences. Most web sites that most people visit are merely serving content. Sites like Facebook and Twitter are pouplar examples—you go to these sides to read what's posted there. Even commerce sites like Amazon primarly serve content to you. You interact in a simple way by clicking buttons to populate a shopping cart or check out.

None of these sites require anything more than what the web's true nature gives us. Yet, interacting with sites like these and others is unpleasant. It is difficult to read the content you are there to read. It's cumbersome to navigate between different areas of the site. On a mobile device, it's worse, since these sites are full of land-mines that respond to a finger-tap and whisk you away from what you are trying to read.

We would benefit if they embraced Brutalist Web Design.

What Can You Do? Link Here

By using semantic HTML and applying zero styling, a web page is emminetly readable and usable, adhering to all the tennets of Brutalist Web Design. If your content is good, your visitors will be very happy.

As you design and build your site, consider the guidelines here. Remove that ridiculous hero image from the top of your page. Don't drop-in a JavaScript email sign-up form. Fight the urge to make your links unadorned, and animate the page when the user clicks on elements. Add styling and decoration to solve a problem only.
